## Affected Pages / Routes

- `/` (PWA) — todo CRUD, offline banner, sync status indicators
- `GET /v1/todos` — list all todos
- `POST /v1/todos` — create todo (with optional client-generated UUID)
- `GET /v1/todos/:id` — find one todo (happy path + 404)
- `PATCH /v1/todos/:id` — update todo (happy path + 404)
- `DELETE /v1/todos/:id` — delete todo (happy path + 404)
- `GET /health` — NestJS health check

## Key Interactions to Verify

### PWA — Offline-first sync (L1 Vitest, L4 Playwright)

- Create todo while offline → item appears in UI immediately with syncStatus: 'pending'
- Reconnect → item flushes to API, syncStatus transitions to 'synced'
- Offline banner appears when navigator.onLine = false; disappears on reconnect
- Error state displayed (not crash) when API returns 4xx/5xx during flush

### PWA — Idempotent flush (L1 Vitest)

- Offline create → disconnect before flush completes → reconnect → item not duplicated in API
- Client-generated UUID preserved in POST body to /v1/todos

### PWA — Partial flush failure (L1 Vitest)

- Queue has 3 items; item 2 fails → items 1 and 3 succeed → item 2 remains in queue
- Next reconnect retries only item 2

### NestJS — CreateTodoDto with client UUID (L3 smoke)

- POST /v1/todos with { id: "client-uuid", description: "test" } → 201, response.id = "client-uuid"
- POST /v1/todos without id → 201, response.id is server-generated UUID

### NestJS — Full CRUD (L3 smoke, real SQLite)

- POST → GET list → assert item present
- POST → GET /:id → assert fields match
- PATCH /:id → GET /:id → assert description/completed updated
- DELETE /:id → GET /:id → assert 404
- POST with empty description → 400 validation error
- GET /health → 200 { status: 'ok' }

### Shared types package — Zod schema validation (L1 Vitest)

- Valid TodoItem payload passes Zod parse
- Invalid payload (missing description, wrong types) fails with correct error
- CreateTodoDto with and without id both pass validation

## Edge Cases

- Queue persists across page reloads (localStorage survives browser refresh while offline)
- Rapid creates while offline → all queued items flush in insertion order on reconnect
- API returns 503 during flush → item stays in queue, error surfaced, no crash
- NestJS DELETE on non-existent id → 404 (not 500)
- NestJS PATCH on non-existent id → 404 (not 500)
- POST /v1/todos with same UUID twice → second write should succeed (idempotent upsert or 409 — verify behavior)

## Critical Paths

1. **Offline create → sync to API:** User creates todo while offline, comes back online, todo persists in API and survives page reload — verifies full offline-first cycle end-to-end
2. **Docker Compose full stack (L4):** `docker compose up`, create todo via PWA UI, todo persists in SQLite after `docker compose down && docker compose up` — verifies nginx proxy, Docker networking, and volume persistence
3. **NestJS smoke test (L3):** POST → GET → DELETE cycle against real SQLite — verifies Prisma migrations ran, queries work, and cleanup is clean between tests
4. **Shared types contract:** Both PWA and NestJS compile without errors with @todo-skeleton/types installed as workspace dependency — verifies the package shape matches both consumers

## Playwright Offline Mechanism

Use `page.context().setOffline(true)` — this sets `navigator.onLine = false` AND fires the `offline` event. Do NOT use `page.route()` alone — it intercepts requests but does not update `navigator.onLine` and will not trigger the sync queue flush handler.
