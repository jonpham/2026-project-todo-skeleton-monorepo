# Idea: Docs-First Workflow (Remove GitHub Issue Sync)

**Status:** Implemented (2026-04-28)
**Driver:** Token cost of GitHub API round-trips in `plan_project`

## Problem

`/project:plan_project` burned excessive tokens creating GitHub issues, labels,
milestones, and project board items — all of which are just a rendering layer on
top of data already in `docs/features/`. Every label, milestone, and board item
required a separate API call.

## Decision

- Feature docs in `docs/features/` are the task management system. Frontmatter
  (`status`, `branch`, `pr`, `completed_at`) and step checklists are the source
  of truth.
- Planning outputs (gstack `/plan-eng-review`, `/autoplan`, GSD plan docs) are
  committed directly to `docs/initiatives/{name}/`.
- GitHub Issues are optional — use manually for external collaboration only.

## What Changed

- `plan_project` and `bootstrap` commands deprecated with warnings pointing to
  gstack / GSD / superpowers equivalents.
- `docs/features/_TEMPLATE.md` deleted — new docs generated by gstack/GSD/superpowers.
- GitHub-specific frontmatter (`issue`, `parent_issue`, `upstream_repos`,
  `upstream_issues`) removed from unworked `[TODO]` feature docs.
- `GSTACK_HOME` set in `.claude/settings.local.json` so gstack writes to the repo
  root (`.gstack/` gitignored). Outputs worth keeping are moved to `docs/initiatives/`.

## Tooling Convention

| Tool        | Output location                          |
| ----------- | ---------------------------------------- |
| gstack      | `docs/initiatives/{name}/`               |
| GSD         | `docs/initiatives/{name}/`               |
| superpowers | `docs/initiatives/{name}/`               |
| Feature docs | `docs/features/[STATUS]GH{n}_{slug}.md` |
| Ideas/ADRs  | `docs/ideas/`                            |
